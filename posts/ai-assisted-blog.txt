<h2 id="从零开始">从零开始</h2></p><p>参考《明日方舟：终末地》官网的暗色工业风设计，用 <strong>Next.js 14</strong> 搭建了一个个人随笔博客，Markdown 写文章，静态生成后部署到 GitHub Pages。
<p>
<h2 id="一路踩过的坑">一路踩过的坑</h2>
<p>
<h3>1. 仓库公开才能用 GitHub Pages</h3></p><p>GitHub Pages 免费版只支持<strong>公开仓库</strong>，私有仓库需要付费。把仓库改为公开后就能用了。
<p>
<h3>2. GitHub Actions 分支策略</h3></p><p>第一次部署报错：
<p>``<code>
<p>Branch "master" is not allowed to deploy to github-pages
</code>`<code>
<p>需要在仓库 Settings → Pages 里正确选择 <strong>GitHub Actions</strong> 作为部署源，而不是默认的 "Deploy from a branch"。
<p>
<h3>3. 部署后 UI 全崩</h3></p><p>推送到 GitHub 后网站样式全没了——白色背景加一堆超链接。原因是 GitHub Pages 项目站点的路径是 </code>/my-blog/<code>，但 Next.js 默认的资源路径是 </code>/_next/static/...<code>，浏览器自然找不到 CSS/JS。</p><p>修复：在 </code>next.config.js<code> 加上 </code>basePath: '/my-blog'<code>。
<p>
<h3>4. Chrome 能打开，Edge 不行（最诡异的坑）</h3></p><p>Edge 会把地址栏输入的 </code>https://air56.github.io/my-blog/<code> 自动替换成 </code>http://myblog.rin/<code>（502 错误）。但如果从<strong>搜索栏</strong>进入却正常。</p><p>排查后发现：之前曾在 GitHub Pages 设置过自定义域名 </code>myblog.rin<code>，虽然删除了配置，但 <strong>Edge 已经缓存了那条 301 永久重定向</strong>，每次输入网址都在本地直接跳转，请求根本没发出去。</p><p>搜索栏能打开，是因为搜索请求经过了搜索引擎中转，形成了新的导航上下文，绕过了本地 301 缓存。</p><p>Chrome 没事，是因为配置 </code>myblog.rin` 时 Chrome 根本没访问过那个网站，压根没缓存那条 301。</p><p>修复：清除 Edge 缓存即可。
<p>
